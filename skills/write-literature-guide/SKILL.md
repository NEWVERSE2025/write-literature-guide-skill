# 文献导读写作

## 概述

将研究论文整理成面向技术读者的中文文献导读。导读应保留原文的事实、关键数据、研究方法、图表线索和工程意义，并写成凝练、规范的中文学术说明文。

开始撰写前，读取 `references/guide-rules.md`，其中包含由样本库归纳出的栏目模板、写作风格和质检清单。

## 工作流程

1. 盘点用户提供的文件，将其区分为原始论文、中文译文、导读样例、文字笔记或重复文件。
2. 按以下优先级阅读材料：已有导读或文字笔记；论文题名、摘要和结论；研究方法与结果章节；图题和表题；中文译文中的术语表达。
3. 提取事实简报：
   - 期刊、题名、作者、机构、DOI、年份、卷期页等论文信息
   - 研究对象、研究核心、研究方法、工程场景、数据集或试验条件
   - 研究问题、方法设计、验证证据、定量发现、局限性和工程价值
   - 值得列入导读的代表性图题和表题
4. 按固定顺序起草：
   `【文献导读】...`、`论文信息`、`摘要`、`研究背景`、`研究方法`、`结果`、`结论`、`引用`。
5. 按样本风格润色：段落紧凑，技术动词明确，结果和结论可使用“标签：内容”的写法，必要时保留中文术语后的专业缩写，避免无依据扩展。
6. 定稿前核对所有数字、方法名、规范名、材料、地点和引用信息，确保均可从源文献中找到依据。

## 输出规则

- 用户要求直接给文本时，按相同标题结构输出中文导读。
- 用户要求生成文档时，创建 `.docx` 文件，并按 `references/guide-rules.md` 中的“DOCX 版式规则”套用模板库格式。
- 源文献信息缺失时，不要编造；使用可核实的最具体信息，无法确认的字段可以省略。
- 作者和期刊名称应忠实于源文献；机构名称只有在源文献或译文支持时才翻译成中文。
- 不要加入教程式说明、个人评论或与本文理解无关的泛泛综述。
